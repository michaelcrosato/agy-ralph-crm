# Specification: Campaign ROI & Performance Analytics API - Brief

## 1. Functional Objective
In standard enterprise CRMs, tracking marketing spend effectiveness is crucial. While we can associate leads and contacts with campaigns and opportunities via Campaign Influence, there is currently no unified core service or REST endpoint that consolidates these records to evaluate a Campaign's exact Return on Investment (ROI) and performance metrics.

This feature introduces the Campaign ROI & Performance Analytics API. The engine will calculate:
- **Total Members Count**: Total number of leads and contacts linked to the campaign.
- **Responded Members Count**: Number of campaign members with status `"Responded"`.
- **Won Opportunities Count**: Number of opportunities influenced by the campaign that have been won (stage `"Closed Won"`).
- **Won Revenue Share Sum**: The consolidated total financial value generated by the campaign, calculated as the sum of `revenueShare` from won opportunities influenced by it.
- **Net Value**: The difference between the Won Revenue Share Sum and the campaign's actual cost (`wonRevenueShareSum - actualCost`).
- **ROI**: The percentage rate of return calculated as `(netValue / actualCost) * 100`. If actual cost is zero, ROI defaults to `0.0`.

All calculations must run under strict Row-Level Security (RLS) isolation per tenant.

## 2. Technical Scope
- **Core Analytics Engine**: Implement a `calculateCampaignROI` core function in `packages/core` that consolidates campaign cost, campaign members status, and campaign influence share records to compute ROI and performance metrics.
- **REST Endpoints**: Expose a REST route `GET /api/campaigns/:id/roi` inside `apps/api` returning the consolidated ROI breakdown.
- **Tenant RLS & Security**: Ensure the campaign and all related records (influences, opportunities, members) are queried under the active tenant's context. An organization must not be able to query another's campaign ROI.
- **Verification & Integration Tests**: Write full integration tests inside `packages/testing/src/campaign-roi.test.ts` validating correct mathematical computations and strict multi-tenant isolation.
