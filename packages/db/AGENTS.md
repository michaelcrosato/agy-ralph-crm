# Database & Persistence Constraints (AGENTS.md)

This path-scoped file outlines execution guidelines and constraints for the Drizzle / PostgreSQL persistence engine.

## Local Constraints
- **RLS Tenancy Enforcement:** Every database access route MUST set the transaction session variable `app.current_org_id` before querying. Raw non-RLS database bypasses are completely prohibited.
- **Migration Protocol:** All database mutations must be defined via Drizzle migration files generated by `pnpm db:generate`. Direct manual schema updates on PostgreSQL instances are barred.
- **Query Complexity Budget:** Avoid deeply nested relational joins in a single query. Compile and run critical performance budget audits using `turbo run test`.
- **Property-based Tenant Separation Tests:** Ensure property-based tenant security tests in `packages/testing` are fully updated and executed to verify database multi-tenant isolation policies.
